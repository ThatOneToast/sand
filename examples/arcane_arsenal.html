Arcane Arsenal

A complete magic RPG datapack written in Rust — custom weapons, mana HUD, spells, loot, recipes, advancements, and a full resource pack.

⚗️ Mana System ⚔️ Custom Weapons ✨ Spell System 🎨 Resource Pack 📜 Advancements 🧪 Loot Tables

🗺️ Project overview

Arcane Arsenal turns vanilla Minecraft into a magic RPG. Players discover and craft powerful weapons, spend mana on abilities, and progress through a spell unlock system. Every piece is defined in Rust using Sand's typed API — the compiler catches mistakes before they ever reach the game.

⚗️

Mana System

Per-player scoreboard mana with tick-based regeneration and a pixel-perfect HUD bar.

⚔️

Custom Weapons

Void Blade, Thunder Staff, and Inferno Wand with typed attributes and model data.

🧙

Spell System

Five unique spells with execute chains, particles, sounds, titles, and cooldowns.

🛡️

Armor Events

Equip/unequip detection for Arcane Robes with automatic buff application.

📜

Advancements

Progressive spell unlock tree with display tabs, criteria, and reward functions.

⚙️ sand.toml config

A single sand.toml controls the datapack namespace, description, target version, and resource pack settings. The pack_format is derived automatically from the version but can be overridden.

sand.toml
[pack]
namespace   = "arcane_arsenal"
description = "Arcane Arsenal — A magic RPG datapack for Minecraft 1.21.4"
mc_version  = "1.21.4"
# pack_format is derived automatically (61 for 1.21.4); uncomment to override:
# pack_format = 61

[resourcepack]
description          = "Arcane Arsenal — HUD bars, spell overlays, and custom item textures"
# resource_pack_format derived automatically (46 for 1.21.4); uncomment to override:
# resource_pack_format = 46

🚀 Initialization

The #[component(Load)] macro registers a function into minecraft:load — it runs once every time the datapack is (re)loaded. This is where all scoreboards, storage defaults, and server-side state are created.

💡

scoreboard objectives add is idempotent in Minecraft — calling it on an already-existing objective is a no-op, so it's safe to run every reload without resetting player data.

src/lib.rs — on_load
use sand_core::{cmd::*, mcfunction};
use sand_macros::{component, function};

/// Runs once every time the datapack is (re)loaded.
///
/// Creates all scoreboards, initializes NBT storage defaults, and applies
/// initial state to every online player via [`init_player`].
#[component(Load)]
pub fn on_load() {
    mcfunction! {
        // ── Mana system ──────────────────────────────────────────────────────
        "scoreboard objectives add arcane_mana dummy";
        "scoreboard objectives add arcane_max_mana dummy";
        "scoreboard objectives add arcane_regen_timer dummy";

        // ── Cooldown objectives (one per spell) ──────────────────────────────
        "scoreboard objectives add cd_void_strike dummy";
        "scoreboard objectives add cd_thunder_bolt dummy";
        "scoreboard objectives add cd_inferno_wave dummy";
        "scoreboard objectives add cd_frost_nova dummy";
        "scoreboard objectives add cd_arcane_shield dummy";

        // ── Stat tracking ────────────────────────────────────────────────────
        "scoreboard objectives add arcane_kills dummy";
        "scoreboard objectives add arcane_spells_cast dummy";

        // ── Regen timer (counts server ticks, resets at 20) ──────────────────
        "scoreboard objectives add regen_tick dummy";

        // ── Apply initial values to all currently online players ─────────────
        "execute as @a run function arcane_arsenal:init_player";

        // ── Announce reload ──────────────────────────────────────────────────
        r#"tellraw @a {"text":"[Arcane Arsenal] ","color":"dark_purple","extra":[{"text":"Datapack loaded.","color":"gray"}]}"#;
    }
}

/// Initializes a single player's mana, max mana, and regen timer.
///
/// Called by [`on_load`] for all online players and by the join advancement
/// for new players. Uses `if score` to skip already-initialized players.
#[function]
pub fn init_player() {
    mcfunction! {
        // Set max mana to 100 if not already set (0 means uninitialized)
        Execute::new()
            .if_score_matches("@s", "arcane_max_mana", "0")
            .run("scoreboard players set @s arcane_max_mana 100");

        // Set current mana to max if uninitialized
        Execute::new()
            .if_score_matches("@s", "arcane_mana", "0")
            .run(Execute::new()
                .store_result_score("@s", "arcane_mana")
                .run("scoreboard players get @s arcane_max_mana"));

        // Reset regen timer
        Execute::new()
            .if_score_matches("@s", "regen_tick", "0")
            .run("scoreboard players set @s regen_tick 20");
    }
}

⚗️ Mana system

The mana system uses three scoreboards and a tick-based regen loop. Every 20 ticks (1 second) each player with less-than-max mana regenerates 5 mana. The current mana value is shown in the HUD via the resource pack.

src/lib.rs — tick & mana regen
/// Main tick function — runs every server tick for every player.
///
/// Delegates to mana regen, cooldown ticking, and HUD updates.
/// Registered into `minecraft:tick` via `#[component(Tick)]`.
#[component(Tick)]
pub fn tick() {
    mcfunction! {
        // Tick all cooldown scores down toward 0 for every player
        Execute::new()
            .as_(Selector::all_players())
            .if_score_matches("@s", "cd_void_strike", "1..")
            .run("scoreboard players remove @s cd_void_strike 1");

        Execute::new()
            .as_(Selector::all_players())
            .if_score_matches("@s", "cd_thunder_bolt", "1..")
            .run("scoreboard players remove @s cd_thunder_bolt 1");

        Execute::new()
            .as_(Selector::all_players())
            .if_score_matches("@s", "cd_inferno_wave", "1..")
            .run("scoreboard players remove @s cd_inferno_wave 1");

        Execute::new()
            .as_(Selector::all_players())
            .if_score_matches("@s", "cd_frost_nova", "1..")
            .run("scoreboard players remove @s cd_frost_nova 1");

        Execute::new()
            .as_(Selector::all_players())
            .if_score_matches("@s", "cd_arcane_shield", "1..")
            .run("scoreboard players remove @s cd_arcane_shield 1");

        // Advance the regen timer for all players
        "scoreboard players add @a regen_tick -1";

        // Trigger mana regen for any player whose timer hit 0
        Execute::new()
            .as_(Selector::all_players().tag("arcane_initialized"))
            .if_score_matches("@s", "regen_tick", "..0")
            .run("function arcane_arsenal:mana_regen");

        // Update HUD for all arcane players
        Execute::new()
            .as_(Selector::all_players().tag("arcane_initialized"))
            .run("function arcane_arsenal:update_hud");
    }
}

/// Regenerates 5 mana for the executing player (run as @s).
///
/// Clamps mana at max mana and resets the regen timer to 20 ticks.
/// Only runs for players with `regen_tick` ≤ 0.
#[function]
pub fn mana_regen() {
    let regen_amount = 5_i32;

    mcfunction! {
        // Add regen_amount mana
        format!("scoreboard players add @s arcane_mana {regen_amount}");

        // Clamp to max: if current > max, set current = max
        Execute::new()
            .store_result_score("@s", "arcane_mana")
            .if_score_compare("@s", "arcane_mana", ">", "@s", "arcane_max_mana")
            .run(Execute::new()
                .store_result_score("@s", "arcane_mana")
                .run("scoreboard players get @s arcane_max_mana"));

        // Reset regen timer
        "scoreboard players set @s regen_tick 20";
    }
}

/// Attempts to consume `cost` mana from the executing player.
///
/// Returns true (sets `arcane_result 1`) if enough mana was available.
/// Returns false (sets `arcane_result 0`) and plays a failure sound if not.
/// Callers check `arcane_result` after calling this function.
#[function]
pub fn try_spend_mana_40() {
    let cost = 40_i32;
    mcfunction! {
        // Default: failure
        "scoreboard players set #result arcane_result 0";

        Execute::new()
            .if_score_matches("@s", "arcane_mana", format!("{cost}.."))
            .run(format!("scoreboard players remove @s arcane_mana {cost}"));

        Execute::new()
            .if_score_matches("@s", "arcane_mana", format!("{}..999", 0))
            .run("scoreboard players set #result arcane_result 1");

        // Play failure sound if not enough mana
        Execute::new()
            .if_score_matches("#result", "arcane_result", "0")
            .run(Sound::play("minecraft:block.glass.break")
                .to(Selector::self_())
                .source(SoundSource::Player)
                .volume(0.8)
                .pitch(0.5)
                .build());
    }
}

🛡️ Custom armor & equip events

Sand's armor event system auto-generates a tick-based detection function that fires when a player equips or unequips a specific item. No manual tick polling needed — just annotate a function with #[component(ArmorEquip(...))] or #[component(ArmorUnequip(...))].

🧙

Under the hood, Sand generates an execute if items entity @s armor.chest … check injected into minecraft:tick. The detection uses a tag to track equipped state — no scoreboard overhead needed.

src/armor.rs
use sand_core::{cmd::*, mcfunction};
use sand_macros::{component, function};

/// The **Arcane Robes** — chest armor that boosts max mana and mana regen.
///
/// - Base item: `leather_chestplate` (dyed dark purple, custom_model_data 3001)
/// - Equipping grants: +50 max mana, +10 regen per second
/// - Unequipping reverts the bonuses automatically
#[component]
pub fn arcane_robes() -> sand_core::components::custom_item::CustomItem {
    use sand_core::components::custom_item::{
        AttributeModifier, AttributeOperation, AttributeType, CustomItem,
        DyedColor, EquipmentSlotGroup, EquippableProperties, EquipmentSlot,
        ItemRarity,
    };
    use sand_core::cmd::generated::Item;

    CustomItem::new("arcane_arsenal:arcane_robes".parse().unwrap(), Item::LeatherChestplate)
        .rarity(ItemRarity::Epic)
        .custom_name(
            r#"{"text":"Arcane Robes","color":"dark_purple","italic":false,"bold":true}"#,
        )
        .lore_line(
            r#"{"text":"Woven from compressed mana crystals.","color":"dark_gray","italic":true}"#,
        )
        .lore_line(
            r#"{"text":"+50 Max Mana  +10 Mana/s","color":"aqua","italic":false}"#,
        )
        .attribute_modifier(
            AttributeModifier::new(AttributeType::Armor, AttributeOperation::AddValue)
                .id("arcane_arsenal:robes_armor")
                .amount(6.0)
                .slot(EquipmentSlotGroup::Chest),
        )
        .attribute_modifier(
            AttributeModifier::new(AttributeType::ArmorToughness, AttributeOperation::AddValue)
                .id("arcane_arsenal:robes_toughness")
                .amount(1.0)
                .slot(EquipmentSlotGroup::Chest),
        )
        .equippable(
            EquippableProperties::new(EquipmentSlot::Chest)
                .equip_sound("minecraft:item.armor.equip_leather")
                .dispensable(true)
                .swappable(true)
                .damage_on_hurt(true),
        )
        .dyed_color(DyedColor::new().hex("#2E0A5B"))
        .custom_model_data(3001)
}

// ── Equip / Unequip event handlers ───────────────────────────────────────────

/// Runs when the player equips Arcane Robes in their chest slot.
///
/// Grants +50 to `arcane_max_mana` and sets the `wearing_robes` tag.
/// Sand auto-generates the detection mechanism — no manual tick check needed.
#[component(ArmorEquip(slot = "chest", item = "arcane_arsenal:arcane_robes"))]
pub fn equip_arcane_robes() {
    mcfunction! {
        // Grant the mana bonus
        "scoreboard players add @s arcane_max_mana 50";
        // Mark the player as wearing robes (used by spell checks)
        "tag @s add wearing_robes";
        // Play an equip sound
        Sound::play("minecraft:item.armor.equip_leather")
            .to(Selector::self_())
            .source(SoundSource::Player)
            .volume(1.0)
            .pitch(1.2)
            .build();
        // Display a subtitle
        Title::of(Selector::self_())
            .subtitle(TextComponent::literal("Arcane Robes equipped — +50 Max Mana")
                .color(ChatColor::DarkPurple))
            .times(0, 30, 10)
            .build();
    }
}

/// Runs when the player unequips Arcane Robes from their chest slot.
///
/// Reverts the +50 max mana bonus and removes the `wearing_robes` tag.
/// Also clamps current mana to the new lower maximum.
#[component(ArmorUnequip(slot = "chest", item = "arcane_arsenal:arcane_robes"))]
pub fn unequip_arcane_robes() {
    mcfunction! {
        "scoreboard players remove @s arcane_max_mana 50";
        "tag @s remove wearing_robes";
        // Clamp current mana to new max
        Execute::new()
            .store_result_score("@s", "arcane_mana")
            .if_score_compare("@s", "arcane_mana", ">", "@s", "arcane_max_mana")
            .run("scoreboard players get @s arcane_max_mana");
    }
}

✨ Spells & abilities

Each spell is a #[function] that chains together mana cost checks, cooldown guards, Execute builders for targeting, particle effects, sounds, titles, and damage effects. The typed API makes it impossible to produce malformed command strings.

src/spells.rs
use sand_core::{
    cmd::{
        generated::Item, Actionbar, ChatColor, Coord, Execute,
        ParticleEffect, ParticleSpread, Selector, Sound, SoundSource,
        TextComponent, Title, Vec3,
    },
    mcfunction,
};
use sand_macros::function;

// ── Void Strike ───────────────────────────────────────────────────────────────

/// **Void Strike** — instant single-target damage with void visual effects.
///
/// Cost: 30 mana | Cooldown: 3 seconds (60 ticks)
///
/// Effect: deals 8 hearts of damage to the nearest enemy within 10 blocks,
/// applies a 3-second levitation effect, spawns void particle burst,
/// plays a sinister sound, and shows a brief title.
#[function]
pub fn spell_void_strike() {
    mcfunction! {
        // ── Guard: check cooldown ─────────────────────────────────────────────
        Execute::new()
            .unless_score_matches("@s", "cd_void_strike", "0")
            .run(Actionbar::show(
                TextComponent::literal("Void Strike on cooldown!")
                    .color(ChatColor::DarkPurple)
            ));
        Execute::new()
            .unless_score_matches("@s", "cd_void_strike", "0")
            .run("return 0");

        // ── Guard: check mana ─────────────────────────────────────────────────

⏱️ Cooldown system

Sand's Cooldown builder produces idiomatic scoreboard-based cooldown logic. You define the objective name and cooldown length in ticks; it generates the guard, start, tick-down, and ready-check commands.

src/lib.rs — using the Cooldown builder
use sand_core::cmd::Cooldown;

// Create a cooldown descriptor: 60-tick (3s) cooldown on objective "cd_void_strike"
let cd = Cooldown::new("cd_void_strike", 60);

// In on_load — register the objective:
// Outputs: scoreboard objectives add cd_void_strike dummy
cd.register();

// In tick — tick all player cooldowns down:
// Outputs: execute as @a if score @s cd_void_strike matches 1.. run scoreboard players remove @s cd_void_strike 1
cd.tick(Selector::all_players());

// In a spell — guard entry:
// Outputs: execute if score @s cd_void_strike matches 1.. run return fail
cd.guard();

// In a spell — start the cooldown after successful cast:
// Outputs: scoreboard players set @s cd_void_strike 60
cd.start();

// Check if ready (for UI):
// Outputs: execute if score @s cd_void_strike matches 0
cd.is_ready();

🧪 Loot tables

Loot tables control what items drop from entities and containers. Sand's typed API covers pools, entries, conditions (entity properties, kill source, random chance), and functions (set count, set nbt, exploration map).

src/loot.rs
use sand_core::components::loot_table::{
    LootCondition, LootEntry, LootFunction, LootPool, LootTable, LootTableType,
    NumberProvider,
};
use sand_core::cmd::generated::Item;
use sand_macros::component;

// ── Ender Dragon Drop ─────────────────────────────────────────────────────────

/// Loot table for the Ender Dragon, injected via `loot replace entity`.
///
/// - 100% chance: 1× Dragon's Heart (represented as a Nether Star with NBT)
/// - 50% chance: 1–3 Mana Crystals
/// - 30% chance: Void Blade custom item
/// - 10% chance: Arcane Robes custom item
#[component]
pub fn loot_ender_dragon() -> LootTable {
    LootTable::new("arcane_arsenal:ender_dragon_drop".parse().unwrap())
        .loot_type(LootTableType::Entity)
        // Pool 1: guaranteed Dragon's Heart
        .pool(
            LootPool::new()
                .rolls(NumberProvider::constant(1.0))
                .entry(LootEntry::item(Item::NetherStar)
                    .function(LootFunction::SetName {
                        name: r#"{"text":"Dragon's Heart","color":"dark_purple","italic":false}"#.to_string(),
                    })
                    .function(LootFunction::SetCount {
                        count: NumberProvider::constant(1.0),
                    })),
        )
        // Pool 2: mana crystals (Amethyst Shards, 50% chance, 1–3)
        .pool(
            LootPool::new()
                .rolls(NumberProvider::constant(1.0))
                .bonus_rolls(NumberProvider::uniform(0.0, 1.0))
                .condition(LootCondition::RandomChance { chance: 0.5 })
                .entry(LootEntry::item(Item::AmethystShard)
                    .function(LootFunction::SetCount {
                        count: NumberProvider::uniform(1.0, 3.0),
                    })),
        )
        // Pool 3: void blade drop (30% chance)
        .pool(
            LootPool::new()
                .rolls(NumberProvider::constant(1.0))
                .condition(LootCondition::RandomChance { chance: 0.30 })
                .entry(LootEntry::item(Item::WoodenSword)
                    .function(LootFunction::SetCustomModelData { value: 1001 })
                    .function(LootFunction::SetName {
                        name: r#"{"text":"Void Blade","color":"dark_purple","italic":false,"bold":true}"#.to_string(),
                    })),
        )
        // Pool 4: arcane robes (10% chance, only on first kill)
        .pool(
            LootPool::new()
                .rolls(NumberProvider::constant(1.0))
                .condition(LootCondition::RandomChance { chance: 0.10 })
                .condition(LootCondition::EntityProperties {
                    entity: "killer_player".to_string(),
                    predicate: serde_json::json!({
                        "player": {
                            "advancements": {
                                "arcane_arsenal:void_adept": true
                            }
                        }
                    }),
                })
                .entry(LootEntry::item(Item::LeatherChestplate)
                    .function(LootFunction::SetCustomModelData { value: 3001 })
                    .function(LootFunction::SetName {
                        name: r#"{"text":"Arcane Robes","color":"dark_purple","italic":false,"bold":true}"#.to_string(),
                    })),
        )
}

// ── Arcane Chest ──────────────────────────────────────────────────────────────

/// Loot table for the arcane treasure chest hidden in the world.

🔍 Predicates

Predicates are reusable condition JSON files. Sand wraps them as Predicate structs with raw JSON conditions — you can reference them in advancements, loot tables, and execute if predicate checks.

src/predicates.rs
use sand_core::components::predicate::Predicate;
use sand_macros::component;

/// Predicate: passes if the entity is holding a Void Blade in their mainhand.
///
/// Used in `execute if predicate arcane_arsenal:has_void_blade` checks
/// throughout the spell and tick systems.
#[component]
pub fn pred_has_void_blade() -> Predicate {
    Predicate::new("arcane_arsenal:has_void_blade".parse().unwrap())
        .condition(serde_json::json!({
            "condition": "minecraft:entity_properties",
            "entity": "this",
            "predicate": {
                "equipment": {
                    "mainhand": {
                        "items": "minecraft:wooden_sword",
                        "predicates": {
                            "minecraft:custom_model_data": { "floats": [1001.0] }
                        }
                    }
                }
            }
        }))
}

/// Predicate: passes if the executing player has at least 1 mana remaining.
///
/// Used as a fast gate before running any mana-cost logic.
#[component]
pub fn pred_has_mana() -> Predicate {
    Predicate::new("arcane_arsenal:has_mana".parse().unwrap())
        .condition(serde_json::json!({
            "condition": "minecraft:entity_scores",
            "entity": "this",
            "scores": { "arcane_mana": { "min": 1 } }
        }))
}

📦 NBT storage

Sand's Storage builder provides a typed HashMap-style interface over Minecraft's data modify storage commands. Use it for per-player state that doesn't fit scoreboards — complex objects, lists, strings.

src/lib.rs — using Storage
use sand_core::cmd::{NbtValue, Storage};

// Namespace all storage under "arcane_arsenal:"
let store = Storage::new("arcane_arsenal:");

// In on_load: set version metadata
store.insert("meta.version", NbtValue::String("0.1.0".into()));

// In on_player_join: push the player name to a visit log list
store.push("logs.visits", NbtValue::String("@s".into()));

// In a spell function: store the last spell cast timestamp
store.insert(
    "spell_state.last_cast",
    NbtValue::Int(0),   // populated at runtime via 'execute store result'
);

// In tick: read back stored value
store.get("spell_state.last_cast");

⛓️ Execute chains

The Execute builder covers all Minecraft 1.21 subcommands. Chains are constructed method-by-method and emit a single, correctly formatted command string. The .run() terminal accepts anything implementing Display.

src/lib.rs — execute chain examples
use sand_core::cmd::{
    Anchor, Coord, Execute, Selector, Vec3,
    generated::{Block, Item},
};

// ── as + at + if_entity ───────────────────────────────────────────────────────
// Run a spell only if the caster is standing on enchanting table
Execute::new()
    .as_(Selector::all_players().tag("arcane_initialized"))
    .at(Selector::self_())
    .if_block(
        Vec3::rel(0.0, -1.0, 0.0),   // block below the player
        Block::EnchantingTable,
    )
    .run("function arcane_arsenal:grant_leyline_bonus");

// ── positioned + anchored ─────────────────────────────────────────────────────
// Spawn particles at eye level of the caster, facing the same direction
Execute::new()
    .as_(Selector::self_())
    .at(Selector::self_())
    .anchored(Anchor::Eyes)
    .positioned(Vec3::local(0.0, 0.0, 1.5))   // 1.5 blocks ahead
    .run("particle minecraft:soul_fire_flame ~ ~ ~ 0 0 0 0.1 5 force");

// ── store result ──────────────────────────────────────────────────────────────
// Count nearby enemies and store in a score
Execute::new()
    .store_result_score("#nearby_count", "arcane_temp")
    .run(format!(
        "execute as {} run return 1",
        Selector::all_entities().not_player().distance_max(10.0)
    ));

// ── if_items (1.20.5+) ────────────────────────────────────────────────────────
// Only run if the player is holding a Void Blade mainhand
Execute::new()
    .as_(Selector::all_players())
    .if_items(
        "entity @s weapon.mainhand",
        "minecraft:wooden_sword[minecraft:custom_model_data~=1001]",
    )
    .run("tag @s add holding_void_blade");

// ── on + summon ───────────────────────────────────────────────────────────────
// Teleport caster to their own target passenger entity, then summon a familiar
Execute::new()
    .as_(Selector::self_())
    .on("passengers")
    .run("function arcane_arsenal:familiar_teleport");

Execute::new()
    .summon("minecraft:vex")
    .run("function arcane_arsenal:init_familiar");

// ── in (dimension) ────────────────────────────────────────────────────────────
// Teleport the player to an exact nether coordinate
Execute::new()
    .as_(Selector::self_())
    .in_("minecraft:the_nether")
    .run(format!(
        "tp @s {}",
        Vec3::abs(100.0, 64.0, 100.0)
    ));

📊 HUD bars

hud_bar! creates a bitmap-font progress bar. Sand assigns Unicode private-use codepoints automatically, generates the sprite sheet PNG (each frame is a slice of the bar), and injects the font provider into assets/arcane_arsenal/font/default.json.

Bars are displayed via the actionbar using display_commands(frame), where frame is an integer 0..=steps.

src/lib.rs — HUD bars (resource pack)
use sand_macros::{hud_bar, hud_element};

// ── Mana bar (20 steps, programmatic blue/dark-blue gradient) ────────────────

/// Mana bar displayed in the actionbar — 20 segments, blue fill.
///
/// `gen!` generates the sprite sheet PNG at build time:
/// - `fill:` is the color of a filled segment (0xAARRGGBB)
/// - `empty:` is the color of an empty segment
/// Sand assigns codepoints U+E100..U+E114 (20 frames + 1 empty).
hud_bar!(
    name:    "mana",
    texture: gen!(fill: 0xFF5599FFFF, empty: 0xFF1A2A4AFF),
    steps:   20,
    height:  9,
    ascent:  9,
);

/// Health bar — rendered below the mana bar using negative ascent offset.
hud_bar!(
    name:    "health",
    texture: gen!(fill: 0xFFE84040FF, empty: 0xFF3A1010FF),
    steps:   20,
    height:  9,
    ascent:  -2,    // rendered below the mana bar
);

/// XP bar — rendered below the health bar.
hud_bar!(
    name:    "xp",
    texture: gen!(fill: 0xFF80FF40FF, empty: 0xFF1A3A10FF),
    steps:   20,
    height:  5,
    ascent:  -13,
);

// ── Using bars in functions ───────────────────────────────────────────────────

/// Updates the HUD bars for the executing player.
///
/// Computes the frame index from the current/max mana ratio and emits
/// the actionbar display command. Called from the main tick.
#[function]
pub fn update_hud() {
    // Compute mana bar frame: (current_mana * 20) / max_mana → 0..20
    // Uses Execute store + division to stay purely in mcfunction
    mcfunction! {
        // Scale mana to 0..20 range
        Execute::new()
            .store_result_score("@s", "arcane_hud_frame")
            .run("scoreboard players get @s arcane_mana");
        "scoreboard players operation @s arcane_hud_frame *= #20 arcane_const";
        Execute::new()
            .store_result_score("@s", "arcane_hud_frame")
            .run(Execute::new()
                .store_result_score("@s", "arcane_hud_frame")
                .run("scoreboard players get @s arcane_hud_frame"));
        "scoreboard players operation @s arcane_hud_frame /= @s arcane_max_mana";

        // Display frame 0..20 via the generated display_commands
        // (Sand emits one 'execute if score … run' per frame index)
        MANA_BAR.display_commands("@s", "arcane_hud_frame", "arcane_arsenal");
    }
}
